## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering (SQL Injection) | db.py CageCode index DDL | mitigate | The `table_name` is derived from `transform.table_name(date_str)` which produces `SAM_PUBLIC_MONTHLY_YYYYMMDD` from a validated 8-digit date. The date is validated by `datetime.strptime(date_str, "%Y%m%d")` in `extract_date()`. No user input flows into the DDL string. The index name is derived deterministically from the table name. Both are wrapped in `[]` bracket quoting. |
| T-02-08 | Elevation of Privilege | run.bat execution | accept | run.bat runs under the Windows Task Scheduler service account. It invokes `python importer.py` -- same privilege level. No elevation risk beyond the configured service account. |
| T-02-09 | Information Disclosure | Temp files left on failure | accept | Per D-05, temp files are intentionally left for operator debugging. They contain public SAM.gov data (not classified). The temp directory is local to the project folder. |
</threat_model>
